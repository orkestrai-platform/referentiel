# Contributing

Thank you for contributing to the OrkestrAI referential.

## Scope

This repository accepts contributions on:
- referential wording clarity;
- multilingual alignment between FR/EN;
- versioned changelog quality;
- typo fixes and consistency improvements.

## Workflow

1. Fork the repository.
2. Create a branch from `main`.
3. Make focused changes.
4. Open a pull request.

## Pull request rules

- One clear topic per PR.
- Describe rationale and expected impact.
- Keep FR and EN aligned when relevant.
- Do not force-push over review feedback without explanation.

## Editorial constraints

- Keep normative language precise and testable.
- Do not introduce ambiguous scoring criteria.
- Preserve backward compatibility for published versions.
- New major semantic changes must target a new version folder (`v2`, etc.).

## Commit messages

Prefer conventional style:
- `docs: clarify criterion A2-C1 wording`
- `docs(fr): fix typo in axis 4`
- `docs(en): align translation with FR source`

## Maintainer validation

All merges are validated by OrkestrAI maintainers according to CODEOWNERS.
